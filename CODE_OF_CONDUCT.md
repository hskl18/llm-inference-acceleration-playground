# Code of Conduct

This project uses a lightweight contributor code of conduct so technical discussion stays useful, specific, and respectful.

## Expected Behavior

- Discuss ideas, measurements, and code with evidence.
- Be clear about assumptions, benchmark limits, and unsupported claims.
- Keep reviews focused on the work and its impact.
- Help new contributors understand project conventions without gatekeeping.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Publishing private information without permission.
- Misrepresenting benchmark results or hiding known measurement limits.
- Repeated disruptive comments that prevent maintainers or contributors from doing useful work.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; block users; or decline contributions when behavior harms the project. Enforcement decisions should be proportional, documented when practical, and focused on preserving a constructive engineering environment.

## Scope

This code applies to project repositories, issue trackers, pull requests, discussions, and related community spaces controlled by the maintainers.
